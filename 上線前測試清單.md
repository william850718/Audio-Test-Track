# AudioTracker 上線前測試清單

> 建議在「測試時段」進行，找 2 個人（或 2 台電腦 / 2 個瀏覽器）一起測。
> 重點是驗證「多人不會互相蓋資料」——這是這次改版的核心。

## A. 環境設定（先做，一次性）
- [ ] 在 Supabase SQL Editor 執行 `aclab-normalize-setup.sql`，出現 Success 無紅色錯誤
- [ ] 在 Supabase SQL Editor 執行 `aclab-storage-setup.sql`（截圖功能用）
- [ ] Database → Replication 看得到 `records` / `projects` / `logs` 三張表
- [ ] Storage 看得到名為 `attachments` 的 bucket，且為 Public

## B. 資料搬遷（第一次登入）
- [ ] 用你的帳號登入，右上角同步狀態顯示 **Synced**（不是紅色 Sync failed）
- [ ] 舊的 Platform 記錄、專案、Log **全部都在**，數量與內容正確
- [ ] 到 Supabase → Table Editor 打開 `records` / `projects` / `logs`，確認裡面有資料
- [ ] 舊的 `app_snapshot` 表仍在（備份，沒被刪）

## C. 單人基本功能（一個人操作即可）
### Platform Tracking
- [ ] 新增一筆記錄 → 出現在清單，右上角 Synced
- [ ] Record Type 選 **Note** → 出現 Sample Type 下拉，可選 6 種樣品
- [ ] 上傳截圖：多張、單檔正常；**總量超過 5MB 會被擋下並提示**
- [ ] 刪除有截圖的記錄後，Supabase Storage 的 `attachments` 裡對應圖片也被清掉（不留孤兒檔）
- [ ] 截圖在卡片上**直接顯示**（不用點開），點一下可放大
- [ ] 編輯記錄、切換狀態(In progress↔Closed)、刪除記錄 都正常
- [ ] 卡片顯示「最後修改者」是你的名字
### Project Log
- [ ] 新增專案 → 進入專案 → 新增 Daily Log（含截圖）→ 正常顯示
- [ ] 編輯 / 刪除 Log、關閉 / 重開 / 刪除專案 都正常

## D. 多人協作（核心！至少 2 人同時）
- [ ] **不互相覆蓋**：A、B 兩人同時開著。A 新增一筆記錄，B 幾秒後畫面**自動出現**那筆（不用重整）
- [ ] 反向再測一次：B 新增、A 自動看到
- [ ] **舊 bug 已解**：A 開著舊畫面 → B 新增一筆 → A 再改自己的別筆記錄並存 → **B 那筆不會消失**
- [ ] **不撞號**：A、B 幾乎同時各新增一筆同平台記錄 → 兩筆都在、ID 不重複（例如一個 TEAMS-005、一個 TEAMS-006）
- [ ] 同一筆記錄 A、B 同時改 → 後存的內容勝出（可接受），但**其他記錄不受影響**

## E. 離線 / 例外
- [ ] 關掉網路後操作 → 顯示「Saved locally, cloud failed」，資料仍在畫面上
- [ ] 恢復網路、重新整理 → 資料正常（以雲端為準）
- [ ] Export 下載 JSON 備份正常；Import 還原正常

## F. 手機
- [ ] 用手機開一次，登入、看記錄、上傳截圖、版面正常

---
若 D 區全部通過 = 多人安全，可以全隊上線。
任何一項紅色錯誤或資料異常，先別上線，把畫面/錯誤訊息回報給我。
