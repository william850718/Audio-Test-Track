# AudioTracker 上線前測試清單

> 建議在「測試時段」進行，找 2 個人（或 2 台電腦 / 2 個瀏覽器）一起測。
> 重點是驗證「多人不會互相蓋資料」——這是核心。

## A. 環境設定（先做，一次性）
- [ ] Supabase SQL Editor 執行 `aclab-normalize-setup.sql`，Success 無紅色錯誤
- [ ] 執行 `aclab-storage-setup.sql`（截圖功能）
- [ ] 執行 `aclab-settings-setup.sql`（Test Software 清單）
- [ ] 執行 `aclab-push-setup.sql`（推播訂閱表）
- [ ] Database → Replication 看得到 `records` / `projects` / `logs` / `app_settings`
- [ ] Storage 看得到 `attachments` bucket，且為 Public

## B. 資料搬遷（第一次登入）
- [ ] 登入後右上角同步狀態顯示 **Synced**（不是紅色 Sync failed）
- [ ] 舊的記錄 / 專案 / Log **全部都在**、數量正確
- [ ] Table Editor 裡 `records` / `projects` / `logs` 有資料
- [ ] 舊的 `app_snapshot` 表仍在（備份，沒被刪）

## C. 單人基本功能
### Platform Tracking
- [ ] 新增記錄 → 出現在清單、Synced；存檔後跳「Record added」提示
- [ ] Record Type 選 **Note** → 出現 Sample Type，可**複選**多種樣品
- [ ] Record Type 選 **Update** → Update Type 選 **Firmware** → 出現 **Test Software** 下拉（含 Other）
- [ ] Test Software 旁「⚙ manage」可新增/刪除軟體，且會雲端同步
- [ ] 上傳截圖：多張、大圖自動壓縮、**總量超過 5MB 被擋**；卡片直接顯示、點擊放大
- [ ] 刪除有截圖的記錄後，Storage 的 `attachments` 對應圖片也被清掉
- [ ] **卡片操作**：滑鼠移到卡片上，右上角浮現圖示 → Edit / Duplicate / Close / Delete 都正常
- [ ] **Duplicate**：以某筆為範本開新增表單（帶入平台/類別/軟體，清空標題與截圖）
- [ ] 卡片版面：標題一行、徽章一列、狀態正常；顯示「最後修改者」
### Project Log
- [ ] 新增專案 → 新增 Daily Log（含截圖）→ 正常顯示
- [ ] 編輯 / 刪除 Log、關閉 / 重開 / 刪除專案 都正常

## D. 篩選・排序・儀表板・匯出
- [ ] 搜尋框打字順暢（debounce，不頓）
- [ ] 篩選：All Categories / Types / Status / **Sample Types** 可疊加
- [ ] 排序（右側 ⇅ 圖示）：Newest / Oldest / 依更新 / 截止日 / 標題 / 平台，切換正確
- [ ] **同一天**建立的多筆記錄，Newest 依序正確（新的在前，不亂）
- [ ] Dashboard：Total / In Progress / Closed / 平均天數 + 長條圖；點卡片可篩選
- [ ] Export：Platform 與 Project 各能匯出 **Excel(CSV)** 與 **PDF**（含 Test Software 欄）
- [ ] 淺色 / 深色切換（右上角 ☀/🌙）：兩種主題文字都清楚、按鈕顏色正常

## E. 多人協作（核心！至少 2 人同時）
- [ ] **不互相覆蓋**：A、B 同時開著。A 新增，B 幾秒後**自動出現**（不用重整）
- [ ] **舊 bug 已解**：A 開著舊畫面 → B 新增 → A 改別筆並存 → **B 那筆不會消失**
- [ ] **不撞號**：A、B 幾乎同時各新增同平台記錄 → 兩筆都在、ID 不重複
- [ ] 同一筆兩人同時改 → 後存的勝出（可接受），其他記錄不受影響

## F. 離線 / 例外
- [ ] 斷網操作 → 狀態顯示「Offline — N pending sync」，資料仍在畫面上
- [ ] 恢復網路（或重新登入）→ 待送的變更**自動補送**、狀態回 Synced
- [ ] Export 下載 JSON 備份正常；Import 還原正常

## G. 推播通知（需先完成 Supabase 推播設定 + VAPID 金鑰）
- [ ] index.html 的 `VAPID_PUBLIC_KEY` 已填、`sw.js` 已在網站根目錄
- [ ] 右上角 🔔 → 允許通知 → 跳「Notifications enabled」
- [ ] 另一人（或另一裝置）新增一筆記錄 → 幾秒內收到系統通知
- [ ] 點通知 → 開啟/切回 AudioTracker
- [ ] iPhone：需先「加入主畫面」當 App 用，才收得到

## H. 手機
- [ ] 手機登入、看記錄、上傳截圖、版面正常
- [ ] 卡片操作圖示在手機上常駐可按（無 hover 也能用）
