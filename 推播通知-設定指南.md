# AudioTracker — Web Push 通知設定指南

當有人在 Platform Tracking 新增一筆記錄時，會即時推播系統通知給「已開啟通知」的所有裝置（即使沒開網頁）。

前置：程式碼都已寫好（`sw.js`、`index.html` 的訂閱程式、`aclab-push-setup.sql`、`supabase/functions/notify-push/index.ts`）。以下是你要做的部署步驟。

---

## 步驟 1 — 產生 VAPID 金鑰（一次）
在有 Node 的電腦執行：
```
npx web-push generate-vapid-keys
```
會得到 **Public Key** 和 **Private Key**，先留著。

## 步驟 2 — 把 Public Key 填進 index.html
打開 `index.html`，找到這一行：
```js
const VAPID_PUBLIC_KEY=''; // <-- paste your VAPID PUBLIC key here...
```
把公鑰貼進去：
```js
const VAPID_PUBLIC_KEY='你的PublicKey';
```

## 步驟 3 — 部署 sw.js 到 GitHub Pages
把 `sw.js` 跟 `index.html` 放在**同一個資料夾**（GitHub repo 根目錄），一起 push 上去。
（`sw.js` 必須跟網頁同源、同一層或更上層，通知才收得到。）

## 步驟 4 — 建立 Supabase 資料表
Supabase → SQL Editor → 貼上 `aclab-push-setup.sql` → Run。

## 步驟 5 — 部署 Edge Function
方法 A（用 Supabase CLI，建議）：
```
supabase functions deploy notify-push
```
方法 B（不想裝 CLI）：Supabase 後台 → Edge Functions → Create function → 命名 `notify-push` → 把 `supabase/functions/notify-push/index.ts` 內容貼進去 → Deploy。

## 步驟 6 — 設定 Function 密鑰（Secrets）
Supabase → Edge Functions → notify-push → Secrets，加入：
| 名稱 | 值 |
|---|---|
| `VAPID_PUBLIC_KEY` | 步驟 1 的 Public Key |
| `VAPID_PRIVATE_KEY` | 步驟 1 的 Private Key（保密！）|
| `VAPID_SUBJECT` | `mailto:你的email@company.com` |
| `APP_URL` | 你的 GitHub Pages 網址，例如 `https://帳號.github.io/repo/` |

（`SUPABASE_URL`、`SUPABASE_SERVICE_ROLE_KEY` 系統會自動提供，不用手動設。）

## 步驟 7 — 建立 Database Webhook
Supabase → Database → Webhooks → Create：
- Table：`records`
- Events：勾 **Insert**（只在「新增記錄」時通知；要連「更新」也通知再加勾 Update，但會比較吵）
- Type：**Supabase Edge Functions** → 選 `notify-push`
- Method：POST

## 步驟 8 — 每個人開啟通知
上線後，每位使用者在工具右上角點 **🔔** → 瀏覽器跳出「允許通知」→ 按允許。之後該裝置就會收到。

---

## 重要限制
- **iPhone / iOS**：必須先把網站「**加入主畫面**」（當 App 用）才收得到 Web Push；純 Safari 開網頁不行。
- **每台裝置、每個瀏覽器**都要各自點一次 🔔 允許通知。
- 沒允許通知的人不會收到（但工具本身照常運作）。
- 通知只在「新增記錄」時發（可在步驟 7 調整）。

## 測試
1. A 裝置點 🔔 允許通知。
2. B（或同一人另開）新增一筆 Platform Tracking 記錄。
3. A 幾秒內應收到系統通知「AudioTracker — new record」。
   - 沒收到 → 看 Supabase → Edge Functions → notify-push → Logs 是否有錯（常見：VAPID 密鑰沒設、webhook 沒指到 function）。
